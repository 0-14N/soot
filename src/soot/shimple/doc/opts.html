Shimple Options

Navindra Umanee

This document describes the command-line options to Shimple. Values for options are specified on the Soot command-line, following the switch -p simple. For example:

java soot.Main -p shimple scalars-only -shimple hello

Shimple is still under active development, so these options are subject to change. For the most current, automatically generated version of this document, please see the file src/soot/shimple/doc/opts.ps in your Soot directory.

General Options

Option optimize

  • Allowed values: true false
  • Default value: false
When this option is set to true, Shimple-based optimizations are applied.

Options for Leaving Shimple

Option naive-phi-elimination

  • Allowed values: true false
  • Default value: false
If set to true, neither pre-optimization nor post-optimization will be applied to the Phi elimination process. Note that setting this option to true takes precedence over other Phi elimination options.

Option pre-optimize-phi-elimination

  • Allowed values: true false
  • Default value: false
If set to true, some optimizations are applied before Phi nodes are eliminated.

Option post-optimize-phi-elimination

  • Allowed values: true false
  • Default value: true
If set to false, optimizations are not applied after eliminating Phi nodes.
